# クラウド同期の修正（容量上限の撤廃）

## 何が問題だったか
旧 Apps Script は進捗 JSON を **1 セル**に保存していた。Google スプレッドシートの
「1 セル = 50,000 文字」上限に達すると（実測 ≈ 57,583 字 / 約 650〜720 問）
書き込みが**静かに失敗**し、それ以降の学習結果が同期されなくなっていた。
さらにクライアントは応答を検証せず常に「同期しました ✓」を表示していたため気づけなかった。

## 修正方針
- **サーバ（GAS）**: `sync.gs` … JSON を複数セルに分割保存（実質サイズ無制限・最大約2MB）。
  応答を正しい JSON (`{ok:true}`) で返すのでクライアントが成否を判定できる。
- **クライアント（3アプリ）**: すでに修正済み（本リポジトリ）
  - `doPush`: 応答を検証。失敗時は正直に警告（端末には保存済み＝データは消えない）。
  - `cloudPull`: **非破壊マージ**（問題ごとに最終解答時刻が新しい方を採用）。
    クラウドが空/古くてもローカルを消さない。
  - ストレージキーを `_v4` に更新し、教科ごとに固有の同期枠（`-muki-v4` / `-yuki-v4` / `-kobun-v4`）。
    → 旧データと衝突せず、空き枠から再スタート。

## デプロイ手順（既存URLを維持＝HTML変更不要）
1. 現在の GAS_URL に対応する **Apps Script プロジェクトを開く**。
2. 既存コードを **`sync.gs` の内容で置き換える**。
   - スクリプトがスプレッドシートに紐づいていない（スタンドアロン）場合のみ、
     `getSheet_()` 内の `openById('★…★')` 行のコメントを外しシートIDを入れる。
3. **デプロイ → デプロイを管理 → 既存の「ウェブアプリ」を編集 → 新バージョン → デプロイ**。
   - 「次のユーザーとして実行: 自分」「アクセスできるユーザー: 全員」。
   - ※既存デプロイを**更新**すれば `/exec` URL は変わらないので、HTML 側の `GAS_URL` 変更は不要。
4. 初回実行時に認可ダイアログが出たら許可。
5. 動作確認: ブラウザで `…/exec?userId=test-v4` を開き `{"ok":true,"data":{}}` が返ればOK。

> もし新URLになってしまった場合は、`muki.html` の `const GAS_URL = "…"` を
> 新URLに変更し、`python gen_apps.py` を再実行（有機/高分子へ自動反映）。

## 「最初から解き直す」について
- ストレージキー `_v4` と新しい同期枠により、3アプリとも**空の状態から開始**します。
- 旧シート（`a-mikan2` 等）と、調査で使った捨てキー `claude-sizeprobe-DELETE-ME-mikan2` の行は、
  不要なので削除して構いません（残しても害はありません）。

## 将来の脱GAS移行（提案・今回は未実装）
GAS は「削除・置換せず修正」の方針どおり維持しました。将来さらに堅牢にするなら:
- **Cloudflare Workers + KV**（無料枠・セル上限なし・302リダイレクトが無く低遅延）。
  クライアントの `GAS_URL` を Worker のURLに差し替えるだけで移行可能（API契約は同じ）。
- 代替: Supabase / Firebase（Firestore）。いずれもサイズ上限の心配がなく、複数端末同期に強い。

実装をご希望なら、Worker のコードと移行手順を別途用意します。
